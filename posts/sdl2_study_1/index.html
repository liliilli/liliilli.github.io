SDL2とは？#

SDL2 公式サイトSDL 2.0のWikiページ

SDLはSimple DirectMedia Layerの略語で、ゲームまたはマルチメディアを応用するアプリケーションを作る時にローレベルの動作をより制御しやすくするために作られたライブラリです。

公式サイトによるとプラットフォームに独立的でWindowsのほか、macOSとかLinux、Androidでも適用できるそうです。

SDL自体はC言語で書かれており、C++またはPython、他Rustなどの様々な言語に比較的移植しやすい特徴を持ってます。

この勉強会ではRust言語で使いやすくしたRust-SDL2を使い、各種機能を勉強して行こうかと考えてます。

Rust-SDL2#

レポジトリー

Rust-SDL2はSDL2のCコードをRust言語でラッピングして、Rustコード上で扱いやすくしたライブラリとなります。

まず最初はRust-SDL2のサンプルコードを書き写ししながらそのコードをCのコードと分析する感じで勉強して行きたいと思います。

SDLの初期化・開放#

SDL Wiki : SDL_Init

まずSDLを使うためにSDLを初期化する必要があります。一番簡単に初期化する方法としては、次のコードとなります。

use sdl2;
let sdl_context = sdl2::init()?;

上の関数を実行することによって、中身ではSdl::new()が走り、基本的にファイル入出力（File I/O）とスレッド（Thread）のサブシステムが一緒に立ち上がります。（かといってスレッドが勝手に生成したりはしないようです。）

C言語のAPIは以下となっていて

int SDL_Init(Uint32 flags);

0以外のフラグを入れたりすることができて、例えば基本的に立ち上がるサブシステム以外のシステムが立ち上上がります。ですがRustでは次のようにサブシステムが初期化できるようです。

let sdl_context: Sdl = sdl2::init()?;
let audio = sdl_context.audio()?;
let event = sdl_context.event()?;
let video = sdl_context.video()?; // videoとeventを同時に立ち上げるそうです。
// ...

またC言語では初期化したSDLコンテキストとサブシステムを明示的に開放する必要がありますが、Rustでは自動でRAIIが行われるので開放する必要がありません。

またRustで立ち上がるSDLコンテキストはThread-safeではないので、そのままでは他のスレッドと共有することはできないみたいです。（中身が Rc<T>でした）むりやり共有するとしたら、MessageQueueを使うか、Arcで包むかですね。

窓の初期化・開放#

SDL Wiki : SDL_CreateWIndow

窓（ウィンドウ）を生成するためにはVideoサブシステムに接近する必要があります。C言語のAPIでは次のSDL_CreateWindow関数で窓に該当するポインターをもらいます。

SDL_Window* SDL_CreateWindow(const char *title, int x, int y, int w, int h, Uint32 flags);

しかしRustライブラリではご定番のビルダーパターン（Builder Pattern）でコンテキストの設定をした後、それをビルドすることで窓のインスタンスがもらえます。

let window = video_subsystem
    .window("ex1_demo", WINDOW_WIDTH, WINDOW_HEIGHT)
    .position_centered()
    .opengl() // あってもいいし、なくてもいいかも
    .build()  // ここで上まで設定した値で窓を生成しようとします。
    .map_err(|e| e.to_string())?;

Wikiを見ればお分かりですが、生成される窓は自動で見せるようになっています。なので、何らかの準備ができてから窓を見せたい場合には

let window = video_subsystem
    .window(..)
    .hidden() // ← THIS
    .build()?;

を使って見せないようにします。その他の高DPI対応のフラグが設定できるメソッドがあったりします。また最初からOpenGL、もしくはVulkanなどの描画コンテキストを初期化させることも可能です。ただ、この場合にはSDL自前のコンテキストを用いることになります。

Rendererの初期化#

SDL Wiki : SDL_CreateRenderer

SDLは3Dの描画は実装してないのですが（もちろんグラフィックAPIを直接使えば3Dまたはコンピューターシェーダーの描画もできるようになるそうです）、基本的に2Dの描画をサポートしています。それをレンダラー（以下、Renderer）と言います。

C言語でRendererを初期化するためにはSDL_CreateRenderer関数にWindowコンテキストを入れて生成します。しかしRustではWindowとRendererが結合してともに管理される形でRendererが取得できます。これを生成するときにもビルダーパターン（Builder）を使ってます。

let mut canvas = window
    .into_canvas()
    .build()
    // …オプションも設定できます。
    .map_err(|e| e.to_string())?;

into_canvas()メソッドは自分を移動するので、一度RendererがビルドされたらWindowコンテキストを持つ変数は使えません。また、ビルドをする時にフラグが設定可能ですが、何もフラグを設定してない場合にはハードウェア加速が基本付きになるらしいです。

このRendererもRustではRAIIがデフォルトなので、明示的に開放する必要はないです。

Rendererを操作してみる#

SDL Wiki : 2D Acceleated Rendering

色々やってみました。

let mut canvas = window.into_canvas().build().map_err(|e| e.to_string())?;
canvas.set_draw_color(Color::RGB(255, 0, 0));
canvas.clear();
canvas.present();

Rendererからのメソッドを使って、描画関数のコール、Viewportの設定、初期値設定などができます。しかし関数のコールをやったとして画面上に描画の内容が即時見れるわけではありません。SDL2は内部でバックバッファ（BackBuffer）を用意しており、::present()を行わないとバックバッファで記録した内容が見せなくなります。

そしてSDL2では簡略なRendererのドライバーの情報が見れます。C言語ではSDL_GetRenderDriverInfoかSDL_GetRendererInfoに相応します。

println!("{:?}", canvas.info());
                                        
for driver in sdl2::render::drivers() {
    println!("{:?}", driver);
}

パソコンの環境によって出力結果が異なりますが、結果を実行したら次のようにSDLでの各種ドライバー、またはコンテキストの情報が見れます。

Eventを使う#

SDL Wiki : Event Handlingrust-sdl2 : Enum sdl2::event::Event

WINAPIなどでもMessageをDispatchして、キー入力を受け取ったり他のものからのメッセージを受け取ったりするように、SDL2でもそういったものに関してはEvent（イベント）を確認する方式を使ってほぼ同じくハンドリングをしています。

SDL2のEventシステムもサブシステムなので、他のサブシステムを一緒に立ち上げる必要がありますが、上でも言ったようにVideoシステムを立ち上げたらこのシステムも同時に立ち上がります。

C言語のコードでイベントを得る（Poll）ためにはSDL_PollEventを使います。Rustではいくつかの方法があって、一つめが関数event::poll_event()を使うか、それともSDLコンテキストから::event_pump()を使うかとなります。

poll_event関数はunsafeであって使いにくいのですが、C言語のAPIそのものにほぼ近いです。::event_pump()を使うと、Rust寄りの使い方となります。こちらでは::event_pump()を使って、再度pumpするまでの間の入力されたイベントリストを参照します。

// このコードは`running loop文で囲まれています。
for event in event_pump.poll_iter() {
    match event {
        Event::Quit { .. } | Event::KeyDown { keycode: Some(Keycode::Escape), ..} 
          => break 'running,
        e => { println!("{:?}", e); }
    }
}

Loopする#

アプリを終了させないためには中身にループを入れる必要があります。しかしループ自体はSDL2でサポートしている機能ではなく、Rustの内蔵ライブラリを使うだけで普通に実装できます。

'running: loop {
    for event in event_pump.poll_iter() {
        // ...
    }
                                                                     
    canvas.clear();
    canvas.present();                                              
    std:: thread::sleep(Duration::from_nanos(1_000_000_000u64 / 30));
}

上のコードでは１秒に30フレーム描画するように書いています。
