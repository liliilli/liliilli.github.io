Window（窓）の操作#

SDL Wiki - Display and Window Management

SDL_GetWindowPosition

SDL2ではVideoシステムから作った窓に関数を使って属性を変えたり内部の数値を取ることができます。例えば次のように現在Windowの位置とサイズを組み合わせて新しい窓のタイトルにすることができます。

let window = canvas.window_mut();
let position = window.position();
let size = window.size();
let title = format!(
    "{} - pos({}, {}), size({}, {}), tick: {}",
    TITLE, position.0, position.1, size.0, size.1, tick
);
window.set_title(&title).map_err(|e| e.to_string())?;

Rust言語でバインディングしたSDL2ライブラリはほとんどのC言語で書いた関数が扱えますが、SDL_SetWindowDataのように任意の値のポインターを保持させて任意で取り出して使えるようにした関数などは使えません。（危険だからでしょうね）

上で見せたメソッド以外に、最小化したり最大化したり、またSurfaceを使って窓のアイコンを変えたりするメソッドも用意されています。

vulkan（ash）との連携#

Ash - A very lightweight wrapper around Vulkan

SDL Graphics with Vulkan

SDL Wiki - Vulkan Support

いきなりぶっ飛びますが、グラフィックAPIの一つであるvulkanをSDL2に連携して属性や窓などを立ち上げたいと思います。SDL2ではOpenGLだけではなく、vulkanもサポートできるよういくつかのAPIを提供しています。

まずvulkanをRustで使うためにはいくつかの方法があります。

  • vulkan自体がC言語で書いているため、自前でFFIを作ってバインディングして使う。
  • vulkanのラッパーライブラリを使う。
  • vulkanを元にしているが、それとは独立したAPIがある描画ライブラリを使って描画をする。

僕は2番目の方法でvulkanをSDL2にバインディングして動かせます。3番目もいい方法で、代表的なライブラリではgfx-rsなどがありますが、こちらではvulkanの奥底の仕組みを勉強しながら実装を進めたいのでやりません。

ash#

2番目の方法、薄いラッパーを使ってRustのコードでvulkanライブラリを使うためにはashライブラリを使う必要があります。このライブラリは比較的最近に発表されたvulkan 1.2バージョンもサポートしながら、元のC言語に近い水準のラッピングの薄さとして関数や構造体を実装しているため、ほぼC言語でコードを書くような感覚で勉強することができます。

このashライブラリの問題点としては、上記の通りC言語関数と構造体へのラッピングが薄いためほぼ全ての関数がunsafe扱いとなっています。とりあえず、ashの紹介はここまでにしてコードを見させてください。

スタート#

https://github.com/Rust-SDL2/rust-sdl2#vulkan

let sdl2_context = sdl2::init().map_err(|e| EError::Sdl2Error(e))?;
let video_subsystem = sdl2_context.video().map_err(|e| EError::Sdl2Error(e))?;
                                                                               
// Make vulkan context window.
let window = video_subsystem
    .window(WINDOW_NAME, WIDTH, HEIGHT)
    .vulkan()
    .build()
    .map_err(|e| EError::Sdl2WindowBuildError(e))?;

最初はSDL2のビデオシステムを立ち上げ、窓（Window）を生成します。窓を生成する時には必ず.vulkan()を実行して、中身にSDL_VULKAN_INITフラグが入ってvulkanが使えるようにします。

ただ、これだけではまだvulkanが使えません。SDL2では特殊な関数があり、それを使って別途で用意したvulkanライブラリから生成されたVkInstanceタイプの値を入れて、拡張機能（KHR）のSurfaceを取る必要があります。

pub fn vulkan_create_surface(
    &self,
    instance: VkInstance
) -> Result<VkSurfaceKHR, String>

上の関数はRust言語専用となりますが、元SDL2ライブラリでも同一な関数がC言語APIとして用意されています。

Extension（拡張）の確認#

C言語API - vkEnumerateInstanceExtensionProperties

VK_KHR_surface

SDL2とvulkanを使ってVkSurfaceKHRを生成する前に、前提条件としてvulkanのSDK環境から拡張としてVK_KHR_surface（またはVK_KHR_win32_surfaceなどのOS依存拡張）がサポートされているかを確認する必要があります。

SDL2-rustでは.vulkan_instance_extensions()メソッドを使って、現在サポートしている全域拡張機能の名前を取得します。

let vk_extensions = window
    .vulkan_instance_extensions()
    .map_err(|e| EError::Sdl2Error(e))?;
for (i, extension) in vk_extensions.iter().enumerate() {
    println!("Vulkan supported extensions {:2} : {}", i, extension);
}

返された値はRustのstrとして返されるため、CStrを使って変換したり無理やり加工する必要がありません。一旦println!を使って出力してみると、次のように利用できる拡張が表示されます。

Vulkan supported extensions  0 : VK_KHR_surface
Vulkan supported extensions  1 : VK_KHR_win32_surface

VK_KHR_surfaceがなければ、VkSurfaceKHRを取得することはできません。

vulkanバージョンの確認#

こちらではvulkanのバージョン1.2の機能を使いたいと思います。もしサポートできるデバイスが無ければ、この時点でアプリケーションを終了するようにしたいです。バージョンを検証する為にはいくつかの方法が存在します。

pub fn vulkan_get_proc_address_function(&self) -> Result<*const ()>

1つ目はvkGetInstanceProcAddr関数を使って、vkEnumerateInstanceVersionの関数ポインターが取れるかを確認することです。関数のポインターが取れたらvulkanのバージョンは1.0ではないことになり、逆だとvulkanはバージョン1.0しかサポートしないことになります。

let get_instance_proc_addr: vk::PFN_vkGetInstanceProcAddr = unsafe {
    mem::transmute(
        video_subsystem
            .vulkan_get_proc_address_function()
            .map_err(|e| EError::Sdl2Error(e))?,
    )
};

ざっくり言いますと、vkGetInstanceProcAddr関数をRustで使うためには型変換が必要となります。C++のreinterpret_cast<>のような効果で適切な関数ポインターに変換するために、unsafeでmem::transmuteを使いました。

vkGetInstanceProcAddrの詳しい使い方は公式マニュアルか、仕様書をご参考ください。

そしてvkGetInstanceProcAddrからvkEnumerateInstanceVersion関数のポインターを取得します。この関数はVkInstanceハンドルを入れる必要がないため、各自のライブラリからのnullに値する値を入れるだけでサポートできれば取れることになります。

let fn_pointer = {
    let fn_signature_to_check = CString::new("vkEnumerateInstanceVersion")?;
    get_instance_proc_addr(vk::Instance::null(), fn_signature_to_check.as_ptr())
};
if fn_pointer.is_none() {
    eprintln!("Supported vulkan environment seems only support 1.0 version.");
    return Err(SomeError);
}

必要Layerとバージョンの確認#

Vulkan Validation Layers - VK_LAYER_KHRONOS_validation

vkEnumerateInstanceLayerProperties

pub unsafe fn enumerate_instance_layer_properties(&self)

VkLayerProperties

vulkanのAPIを使う時に、基本的な動作以外にも値のチェック、エラー機能などを行うためには別途のレイヤー（Layer）を適用する必要があります。VkInstanceを生成する時に適合なレイヤーの名前を入れることによって、Logging、Validation（エラー検出）などができるようになります。

ここではVK_LAYER_KHRONOS_validationValidationレイヤーをインスタンスを生成する時に一緒に立ち上げます。各レイヤーも実装バージョンがあり、できれば検証する必要があります。

const LAYER_KHRONOS_VALIDATON: &'static str = "VK_LAYER_KHRONOS_validation";

vulkanから現在サポートしているレイヤーの情報を確認する方法としてはvkEnumerateInstanceLayerProperties関数を実行することがありますが、ashライブラリではEntryからヘルパーメソッドを立ち上げてVkLayerPropertiesのリストを簡単に取得することができます。

let vk_entry = unsafe { Entry::new()? };
let layer_properties = vk_entry.enumerate_instance_layer_properties()?;

属性のリストが取得できたら、イテレーションで立ち上げたいレイヤーが存在するかを確認します。VkLayerProperties構造体の詳しい情報は公式マニュアルか仕様書をご参考ください。

let required_layers = [(CString::new(LAYER_KHRONOS_VALIDATON)?, vk::make_version(1, 2, 0))];
let all_supported = required_layers.iter().all(|(name, v)| {
    layer_properties
        .iter()
        .find(|&p| {
            let layer_name = unsafe { CStr::from_ptr(p.layer_name.as_ptr() as *mut _) };
            layer_name.to_string_lossy() == name.to_string_lossy() && *v <= p.spec_version
        })
        .is_some()
});
if !all_supported {
    eprintln!("Failed to satisfy all required layer conditions");
    return Err(SomeError);
}

VkInstanceの生成#

VkInstance Manual Page

vkCreateInstance Manual Page

VkInstanceCreateInfo Manual Page

入れたいレイヤー（Layer）、拡張（Extensions）そして情報をまとめてvkCreateInstanceに該当するメソッドに入れます。ashライブラリではEntry::create_instance(&self, ..)を使います。

let enabled_layer_names = required_layers
    .iter()
    .map(|(name, _)| name.as_ptr())
    .collect::<Vec<_>>();
let enabled_extension_names = vk_extensions
    .iter()
    .map(|name| name.as_ptr()) // ここでは所有権回避のための追加コードは抜きました。
    .collect::<Vec<_>>();
                                                                       
// We have to make application information for create vulkan instance.
let c_app_name = CString::new(WINDOW_NAME)?;
let appinfo_builder = vk::ApplicationInfo::builder()
    .application_name(&c_app_name)
    .application_version(0)
    .engine_name(&c_app_name)
    .engine_version(0)
    .api_version(vk::make_version(1, 2, 0));
                                                                       
let instance_create_info = vk::InstanceCreateInfo::builder()
    .application_info(&appinfo_builder)
    .enabled_layer_names(&enabled_layer_names)
    .enabled_extension_names(&enabled_extension_names);
                                                                       
let instance: Instance = unsafe {
    vk_entry
        .create_instance(&instance_create_info, None)
        .expect("Instance Creation Error.")
};

これでInstance（中にVkInstanceが入っています）が生成されました。

レイヤーのプロパティを出力#

VkLayerProperties Manual Page

上では省略してしまったんですが、enumerate_instance_layer_propertiesメソッドで取得したレイヤーのリストから各レイヤーの情報を参照することもできます。

for (i, layer_property) in layer_properties.iter().enumerate() {
    unsafe {
        println!("Vulkan supported layer property {:2}", i);
        let layer_name = CStr::from_ptr(layer_property.layer_name.as_ptr() as *mut _);
        let layer_name = layer_name.to_string_lossy();
        println!("* Layer name : {}", layer_name);
        let description = CStr::from_ptr(layer_property.description.as_ptr() as *mut _);
        let description = description.to_string_lossy();
        println!("* Description : {}", description);
        let version = layer_property.spec_version;
        println!("* Spec version : {}.{}.{}",
            vk::version_major(version), vk::version_minor(version), vk::version_patch(version));
        println!("* Impl version : {}", layer_property.implementation_version);
        println!("");
    }
}

Physical Deviceのプロパティを出力#

VkPhysicalDevice Manual Page

VkPhysicalDeviceProperties Manual Page

vkGetPhysicalDeviceProperties2 Manual Page

Physical Device（物理デバイス）はvulkanをサポートする物理デバイスを称えていうものとなります。

いきなり詳しく説明するのはしづらいのですが、vulkanから描画またはDispatch（コンピューターシェーダー）を行ったり、リソースの作成・全般の制御行うためには論理デバイス（Logical Device）を作る必要があります。その論理デバイスの値を作る時に必要となるハードウェア的な親となります。

let physical_devices = instance.enumerate_physical_devices()?;
physical_devices.iter().enumerate().for_each(|(_i, pd)| {
    // Get physical device general properties.
    let mut vk11_property = vk::PhysicalDeviceVulkan11Properties::default();
    let mut vk12_property = vk::PhysicalDeviceVulkan12Properties::default();
    let mut properties = vk::PhysicalDeviceProperties2::builder()
        .properties(vk::PhysicalDeviceProperties::default())
        .push_next(&mut vk12_property)
        .push_next(&mut vk11_property)
        .build();
    instance.get_physical_device_properties2(*pd, &mut &mut properties);
                                                                                             
    println!("{:?}", properties.properties);
    println!("{:?}", vk11_property);
    println!("{:?}", vk12_property);
                                                                                             
    // Get queue family properties.
    let queue_families_len = instance.get_physical_device_queue_family_properties2_len(*pd);
    let mut queue_family_props = vec![];
    for _ in 0..queue_families_len {
        queue_family_props.push(vk::QueueFamilyProperties2::default());
    }
    instance.get_physical_device_queue_family_properties2(*pd, &mut queue_family_props);
    for prop in &queue_family_props {
        println!("{:?}", prop);
    }
});

上の関数では二つの情報を出していて、上のロジックは物理デバイス関連の情報を出力しています。vulkan 1.0ならvk::PhysicalDevicePropertiesだけしか使えませんが、vulkan バージョン1.2からはもっと詳しい情報などを取るように構造体をつなげて情報を取ることができるようになりました。

上で取得した情報は下のように出力できました。ashライブラリのPhysicalDevicePropertyの中身のコードを見ると、一応fmt::Displayは実装されているのですが、きれいには出力できないみたいですね…。（とりあえず構造体の一部はJSON文と互換できるように出力されることもあります。）

Physical Device Groupのプロパティの出力#

vkEnumeratePhysicalDeviceGroups Manual Page

VkPhysicalDeviceGroupProperties Manual Page

Device Groupとは、単一のコマンドバッファ（コマンドプールからの）が全てのデバイスで動作ができたり、そしてそれぞれのデバイスがそれぞれのVRAM（メモリ）に接近できるセット（郡）を指します。同じグループに属している物理デバイスは、同一な機能と拡張、そしてプロパティをサポートするように保証されています。

とりあえずこのデバイスグループの情報も取得しちゃいます。

let pd_len = instance.enumerate_physical_device_groups_len()?;
let mut pds = vec![];
for _ in 0..pd_len {
    pds.push(vk::PhysicalDeviceGroupProperties::default());
}
instance.enumerate_physical_device_groups(&mut pds)?;
pds.iter().for_each(|prop| println!("{:?}", prop));

上のコードで下のグループの情報が見れるようになりました。

Surfaceをつけてみよう#

VkSurfaceKHR Manual Page

vkDestroySurfaceKHR Manual Page

pub fn vulkan_create_surface

既存のWindowにvulkanを見せるためにSurfaceを作ります。ですが、実はvulkanでは画面の表示の機能は基本的な機能ではなく、拡張として扱いされています。なので、Surface関連ではKHRまたはEXT、FUCHSIAなどの拡張子がついています。

そして元のC言語のAPI通りだと、Surfaceを生成するための関数は拡張によって全部異なります。（リンク先のSee Alsoをご覧いただくとものすごく多いことがわかります。）ですが、SDL2-rustではプラットフォームによって中身から勝手に判断してOSに合わせたSurface作成関数を実行しようとしてますので、少し扱いやすくなります。

let vk_surface: vk::SurfaceKHR = unsafe {
    let r = window
        .vulkan_create_surface(mem::transmute(instance.handle()))
        .map_err(|e| EError::Sdl2Error(e))?;
    mem::transmute(r)
};

この関数で返される値はashライブラリのSurfaceとは違って、むしろそのSurfaceの中身のvk::SurfaceKHRの相当されます。SDL2ライブラリから返される値はu64ですが、mem::transmuteすることでポインターに切り替えます。（32bit環境ではどうなるかはわかりません。）

vk::SurfaceKHRを開放するためにはSDL2からは開放の関数を用意してないし、そしてRAIIもないので明示的にCの関数ポインターから開放を行うしかありませんでした。

// unsafeです。
let fn_name = CString::new(NAME_VK_DESTROY_SURFACE_KHR)?;
let vk_destroy_surface_khr: PFN_vkDestroySurfaceKHR = {
    mem::transmute(
        vk_entry
            .get_instance_proc_addr(instance.handle(), fn_name.as_ptr())
            .unwrap(),
    )
};
vk_destroy_surface_khr(instance.handle(), vk_surface, ptr::null_mut());

雑ですが、これでvk::SurfaceKHRは開放されます。
